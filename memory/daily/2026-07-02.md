## Aprendizajes clave (para no repetir errores)

### Versiones que funcionan juntas (KMP + Compose + Android)
- **Kotlin 2.2.20** + **Compose Multiplatform 1.10.0** + **AGP 8.13.0** + **Gradle 8.14.3**
- NO usar AGP 9.x con KMP: rompe `androidTarget()` + `com.android.application` ("new DSL"). AGP 9 exige Gradle 9.4.1+ y el plugin `com.android.kotlin.multiplatform.library`.
- NO usar Kotlin 2.4.0 con AGP 8.13: error "Module compiled with incompatible version of Kotlin. binary 2.4.0 expected 2.2.0" al mergear recursos Android.
- minSdk **26** obligatorio si se incluye Netty 4.2 (usa MethodHandle.invokeExact, requiere API 26+). Alternativa: engine CIO en Android.
- `local.properties` -> `sdk.dir` debe llevar **barras normales** (`C:/Users/.../Android/Sdk`); con backslash, AGP falla con "IOException: nombre de archivo incorrecto" en SdkLocator.
- `compose.material3` / `compose.materialIconsExtended` marcan deprecation pero funcionan (suprimir con @Suppress("DEPRECATION")).
- Source set intermedio `jvmMain` compartido android+desktop: el intento con `applyHierarchyTemplate { group("jvm") { withJvm(); withAndroidTarget() } }` NO agrupo bien; solucion final fue **duplicar** el codigo networking (ReceiveServer, TransferClient) en `desktopMain` y `androidMain`. Pendiente refactorizar a source set intermedio correcto.
- Desactivar ProGuard en Compose MP 1.10: `compose.desktop.application.buildTypes.release.proguard { isEnabled.set(false) }` (NO `tasks.matching{}.configureEach{enabled=false}`, que rompe la cadena del distribuible).

### Empaquetado MSI local sin admin
- WiX Toolset requiere admin para instalarse via winget. **Alternativa portable**: descargar `wix314-binaries.zip` de github.com/wixtoolset/wix3/releases, descomprimir en `.tools/wix/` y anadir al PATH. jpackage (JDK 17) lo detecta y genera el MSI. `.tools/` en .gitignore.

### GitHub
- gh CLI: token de GitHub Desktop NO tiene scope `read:org`; hay que hacer `gh auth login --web` (codigo OTC de 8 digitos).
- Para pushear `.github/workflows/*` se necesita scope **workflow** extra: `gh auth refresh --scopes workflow` (otra autorizacion OTC).
- `gradlew` subido desde Windows pierde bit de ejecucion -> exit 126 en runners Linux. Fix: `git update-index --chmod=+x gradlew`.
- Mover un tag a un commit arreglado: `git tag -d v0.1.0 && git tag -a v0.1.0 ... && git push origin v0.1.0 --force`. El workflow de release se redispara solo.

## Conversacion 2 — v0.2.0 (fixes de deteccion mDNS + UI)
- **Tarea:** Bug reportado por usuario tras probar v0.1.0: escritorio no detecta moviles (ni viceversa), PIN gigante/desproporcionado, botones no caben, no se pueden seleccionar dispositivos ni emparejar.
- **Evaluacion:** [DET: 4 | STO: 3] -> HIBRIDA.
- **Resultado:** EXITO. v0.2.0 publicada con APK + MSI.
- **Causas raiz encontradas y fixeadas:**
  1. **mDNS no detecta entre si (bug critico):** `JmDNS.create(InetAddress.getLocalHost())` se ataba a loopback/interfaz virtual en Windows. Fix: helper `bestLanInterface()` elige primera IPv4 UP no-loopback no-virtual (prioriza WiFi). Fallback `JmDNS.create()` (0.0.0.0).
  2. **Desktop arrancaba discovery con IP 127.0.0.1:** Fix: calcular `bestLocalIp()` ANTES de `discovery.start()`.
  3. **Android NsdManager perdia servicios:** `resolveService` concurrente falla (solo 1 a la vez). Fix: cola `pendingResolves` + flag `resolving`, resolver de 1 en 1; `onResolveFailed` ahora reporta (no traga error).
  4. **PIN gigante:** `fontSize` hardcodeado 52sp/64sp. Fix: `MaterialTheme.typography.displaySmall`/`displayMedium`.
  5. **No se podian seleccionar dispositivos:** `DeviceItem`/`DeviceRow` sin `clickable`. Fix: clicable -> preselecciona destino y navega a pestana Enviar.
  6. **Estado destino/PIN se perdia al cambiar pestana:** eran `remember` locales destruidos por `when(tab)`. Fix: mover a `StateFlow` en `AndroidAppState`/`AppState` (`sendTarget`, `sendPin`, `send()`).
  7. **Botones no cabian / teclado tapaba:** Fix: `verticalScroll` + `imePadding` + `fillMaxWidth`.
- **Entregables v0.2.0:**
  - APK: `dist/CompartirArchivos-0.2.0.apk` (15 MB).
  - MSI: `dist/CompartirArchivos-0.2.0.msi` (103 MB).
  - Release: https://github.com/jhonsu01/compartir-archivos-app/releases/tag/v0.2.0
- **Aprendizajes Compose Multiplatform:**
  - `when(tab)` desmonta el componible destino -> todo `remember` local se pierde. Para estado que debe cruzar pestañas: StateFlow en el holder de estado (no remember).
  - En Windows, NsdManager/JmDNS requieren bind a la NIC WiFi real; `getLocalHost()` casi siempre resuelve mal.
  - `NsdManager.resolveService` es de 1-solo-vuelo: si hay varios servicios, encolar.

## Conversacion 3 — v0.3.0 (fix transferencia + carpeta descarga + deteccion)
- **Tarea:** Bug: transferencia mov->desktop fallaba con "Fail to prepare request body... Content-Type: null". Desktop seguia sin detectar al movil. Feature pedida: elegir carpeta de descarga.
- **Resultado:** EXITO. v0.3.0 publicada.
- **Causas raiz y fixes:**
  1. **Transferencia fallaba:** Ktor 3.x NO infiere Content-Type del body; `setBody(obj)` con ContentNegotiation instalado precisa `contentType(ContentType.Application.Json)` explicito en el request. NO usar `defaultRequest { contentType(...) }` (la funcion `defaultRequest` no resuelve en Ktor 3.5 sin plugin extra). Fix: `client.post(url) { contentType(Application.Json); setBody(req) }`.
  2. **Deteccion asimetrica (desktop no ve movil):** `JmDNS.create(iface)` con una sola NIC fallaba en Windows con adaptadores virtuales. Fix: `JmDNS.create()` (0.0.0.0) + sondeo activo `mdns.list(type)` cada 5s en hilo daemon (compensa multicast perdido en WiFi). NOTA: el firewall de Windows bloqueando java.exe es causa comun; documentado en README.
  3. **Feature carpeta de descarga:** Android SAF (URI tree persistida en SharedPreferences, guardar via DocumentFile); Desktop JFileChooser (persistida en ~/.compartirarchivos/download_folder.txt).
- **Aprendizajes Ktor 3.x:** el Content-Type es OBLIGATORIO explicito para que ContentNegotiation seleccione converter; el `defaultRequest` builder no existe por defecto en 3.5 (requiere plugin DefaultRequest).
- **Entregables v0.3.0:** APK 15MB + MSI 103MB en https://github.com/jhonsu01/compartir-archivos-app/releases/tag/v0.3.0

## Conversacion 4 — v0.4.0 (deteccion manual + explorador interno + rediseño UI)
- **Tarea:** Desktop seguia sin ver dispositivos. Boton SAF no hacia nada en dispositivos sin gestor de archivos. Mejorar UI movil (mas intuitiva).
- **Resultado:** EXITO. v0.4.0 publicada. Transferencia movil->desktop confirmada funcionando por el usuario en v0.3.0.
- **Diagnostico deteccion desktop (causa raiz confirmada):** NO es bug de codigo. El firewall de Windows bloquea inbound multicast UDP 5353 hacia el proceso java/CompartirArchivos.exe. Asimetria: el desktop puede ENVIAR multicast (el movil lo ve) pero el firewall bloquea RECIBIR (el desktop no ve al movil). HTTP funciona porque es unicast en otro puerto.
- **Solucion aplicada:** conexion MANUAL por IP:puerto en el desktop (boton "Añadir por IP"). El usuario lee la IP del movil (mostrada en su UI) y la escribe en el desktop. No depende de mDNS ni firewall. Robusto ante cualquier red.
- **Explorador Android (3 vias):** (1) Explorador INTERNO basado en java.io.File sobre Download/DCIM/Pictures/Documents/Music — NO requiere apps externas, funciona en TV/dispositivos cerrados. Permiso READ_MEDIA_* (API 33+) pedido en runtime. (2) SAF (carpeta) como alternativa. (3) "Abrir con otra app" (ACTION_OPEN_DOCUMENT) para elegir archivos sueltos.
- **Rediseño UI movil:** pantalla de inicio con tarjetas grandes (Recibir/Enviar/Carpeta descarga); asistente de pasos guiados para enviar (1.Archivos -> 2.Destino -> 3.PIN+Enviar) con indicador de progreso; bottom nav; pantallas dedicadas Recibir y Carpeta descarga.
- **Aprendizajes:**
  - mDNS en Windows desktop depende de que el firewall permita inbound UDP 5353; NUNCA arreglar por codigo. Solucion producto: fallback por IP manual.
  - Compose MP materialIconsExtended 1.10 NO tiene todos los icons de AndroidX (InsertDriveFile, Description, Article ausentes); usar Send/Folder/etc seguros.
  - Explorador interno java.io.File sobre Environment.getExternalStorageDirectory() requiere READ_EXTERNAL_STORAGE (<=32) o READ_MEDIA_* (33+).
- **Entregables v0.4.0:** APK 15MB + MSI 103MB en https://github.com/jhonsu01/compartir-archivos-app/releases/tag/v0.4.0
