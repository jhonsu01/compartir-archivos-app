## Aprendizajes clave (para no repetir errores)

### Versiones que funcionan juntas (KMP + Compose + Android)
- **Kotlin 2.2.20** + **Compose Multiplatform 1.10.0** + **AGP 8.13.0** + **Gradle 8.14.3**
- NO usar AGP 9.x con KMP: rompe `androidTarget()` + `com.android.application` ("new DSL"). AGP 9 exige Gradle 9.4.1+ y el plugin `com.android.kotlin.multiplatform.library`.
- NO usar Kotlin 2.4.0 con AGP 8.13: error "Module compiled with incompatible version of Kotlin. binary 2.4.0 expected 2.2.0" al mergear recursos Android.
- minSdk **26** obligatorio si se incluye Netty 4.2 (usa MethodHandle.invokeExact, requiere API 26+). Alternativa: engine CIO en Android.
- `local.properties` -> `sdk.dir` debe llevar **barras normales** (`C:/Users/.../Android/Sdk`); con backslash, AGP falla con "IOException: nombre de archivo incorrecto" en SdkLocator.
- `compose.material3` / `compose.materialIconsExtended` marcan deprecation pero funcionan (suprimir con @Suppress("DEPRECATION")).
- Source set intermedio `jvmMain` compartido android+desktop: el intento con `applyHierarchyTemplate { group("jvm") { withJvm(); withAndroidTarget() } }` NO agrupo bien; solucion final fue **duplicar** el codigo networking (ReceiveServer, TransferClient) en `desktopMain` y `androidMain`. Pendiente refactorizar a source set intermedio correcto.
- Desactivar ProGuard en Compose MP 1.10: `compose.desktop.application.buildTypes.release.proguard { isEnabled.set(false) }` (NO `tasks.matching{}.configureEach{enabled=false}`, que rompe la cadena del distribuible).

### Empaquetado MSI local sin admin
- WiX Toolset requiere admin para instalarse via winget. **Alternativa portable**: descargar `wix314-binaries.zip` de github.com/wixtoolset/wix3/releases, descomprimir en `.tools/wix/` y anadir al PATH. jpackage (JDK 17) lo detecta y genera el MSI. `.tools/` en .gitignore.

### GitHub
- gh CLI: token de GitHub Desktop NO tiene scope `read:org`; hay que hacer `gh auth login --web` (codigo OTC de 8 digitos).
- Para pushear `.github/workflows/*` se necesita scope **workflow** extra: `gh auth refresh --scopes workflow` (otra autorizacion OTC).
- `gradlew` subido desde Windows pierde bit de ejecucion -> exit 126 en runners Linux. Fix: `git update-index --chmod=+x gradlew`.
- Mover un tag a un commit arreglado: `git tag -d v0.1.0 && git tag -a v0.1.0 ... && git push origin v0.1.0 --force`. El workflow de release se redispara solo.
